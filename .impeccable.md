## Design Context

### Target Audience
EVE Frontier players who manage alliance treasuries — fleet commanders, treasurers, and council elders. These are power users comfortable with complex interfaces, but they expect information density done right (like a ship's tactical HUD), not cluttered admin panels.

### Use Cases
1. Create and manage an alliance treasury with multi-sig governance
2. Submit, sign, and execute budget proposals
3. Configure an autonomous Policy Agent to auto-approve routine spending
4. Monitor alliance activity (killmails, jump events, audit trail)

### Brand Personality / Tone
**Military command console in deep space.** Cold, precise, authoritative. Think submarine CIC or spacecraft bridge — not flashy, not playful. Information is presented with tactical clarity. Every element serves a function.

### Visual Direction
- **Palette**: Deep space blacks (#0a0e17, #0d1117), muted amber (#c9a227) for warnings/highlights, cold cyan (#3b9ece) for interactive elements, military olive for secondary accents
- **Typography**: Already using "Frontier Disket Mono" (headings) + "Favorit" (body) — keep these, they are from EVE Frontier's official font stack
- **Borders**: Thin 1px lines, subtle glow on focus/active states, no rounded corners (sharp/angular)
- **Cards**: No drop shadows. Thin border with slight opacity. Background barely lighter than page.
- **Motion**: Minimal — data appears, it doesn't bounce. Fade-in only where it communicates state change.

### Differentiator
This is not a DeFi dashboard. This is a **fleet command interface** where every SUI transfer is a tactical decision. The design should make you feel like you're authorizing a military operation, not filling out a form.
